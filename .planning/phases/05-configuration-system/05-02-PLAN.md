```cpp
static struct option longOptions[] = {
    // String settings with required_argument
    {"tab-foreground",        required_argument, nullptr, 0},
    {"tab-background",        required_argument, nullptr, 0},
    {"frame-background",      required_argument, nullptr, 0},
    {"button-background",     required_argument, nullptr, 0},
    {"borders",               required_argument, nullptr, 0},
    {"menu-foreground",       required_argument, nullptr, 0},
    {"menu-background",       required_argument, nullptr, 0},
    {"menu-highlight",        required_argument, nullptr, 0},
    {"menu-borders",          required_argument, nullptr, 0},
    {"frame-thickness",       required_argument, nullptr, 0},
    {"auto-raise-delay",      required_argument, nullptr, 0},
    {"pointer-stopped-delay", required_argument, nullptr, 0},
    {"destroy-window-delay",  required_argument, nullptr, 0},
    {"new-window-command",    required_argument, nullptr, 0},

    // Boolean flags: presence = enable (per D-03)
    {"click-to-focus",        no_argument, nullptr, 0},
    {"raise-on-focus",        no_argument, nullptr, 0},
    {"auto-raise",            no_argument, nullptr, 0},
    {"exec-using-shell",      no_argument, nullptr, 0},

    // Boolean negations: --no-xxx = disable (per D-03)
    {"no-click-to-focus",     no_argument, nullptr, 0},
    {"no-raise-on-focus",     no_argument, nullptr, 0},
    {"no-auto-raise",         no_argument, nullptr, 0},
    {"no-exec-using-shell",   no_argument, nullptr, 0},

    {nullptr, 0, nullptr, 0}
};
```

Implement applyCliArgs():
1. Set optind = 1 to reset parsing
2. Loop with getopt_long(argc, argv, "", longOptions, &optionIndex)
3. On c == -1: break (no more options)
4. On c == '?': print "Try 'wm2-born-again --help' for more information." to stderr, exit(2)
5. On c == 0: match longOptions[optionIndex].name to set Config member:
   - Boolean enable (e.g., "click-to-focus"): set member to true
   - Boolean disable (e.g., "no-click-to-focus"): set member to false
   - String settings (e.g., "tab-foreground"): assign optarg to member
   - Integer settings (e.g., "frame-thickness"): parse with std::stoi, clamp to ranges (same ranges as applyKeyValue: delays 1-60000, thickness 1-50), catch exceptions

For tests, construct mock argc/argv arrays and call applyCliArgs() directly. Example:
```cpp
TEST_CASE("CLI --frame-thickness overrides default", "[config][cli]") {
    Config cfg;
    char* argv[] = { const_cast<char*>("wm2"), const_cast<char*>("--frame-thickness=3"), nullptr };
    cfg.applyCliArgs(2, argv);
    REQUIRE(cfg.frameThickness == 3);
}
```

For precedence test: apply a config file first (via applyFile), then apply CLI args, verify CLI wins.

For unknown option test: capture stderr output and verify exit code. Use a subprocess or Catch2's REQUIRE_THROWS approach. Simplest: test that applyCliArgs calls exit(2) on unknown option using Catch2's CHECK_THROWS_WITH or by testing with a known-good option set and a separate test that validates error behavior via fork/exec.

Note: Since applyCliArgs() may call exit(2) on bad input, CLI error tests should use REQUIRE_THROWS or fork the test process. The simplest approach: test only valid option parsing in-process, and test that invalid options are detected by checking getopt_long's behavior is correctly wired.
