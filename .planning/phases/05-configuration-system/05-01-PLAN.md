```cpp
#pragma once
#include <string>
#include <vector>

struct Config {
    // Colors (tab)
    std::string tabForeground   = "black";
    std::string tabBackground   = "gray80";
    // Colors (frame)
    std::string frameBackground = "gray95";
    std::string buttonBackground = "gray95";
    std::string borders         = "black";
    // Colors (menu)
    std::string menuForeground  = "black";
    std::string menuBackground  = "gray80";
    std::string menuHighlight   = "gray60";
    std::string menuBorders     = "black";
    // Focus policy
    bool clickToFocus = false;
    bool raiseOnFocus = false;
    bool autoRaise    = false;
    // Timing (milliseconds)
    int autoRaiseDelay      = 400;
    int pointerStoppedDelay = 80;
    int destroyWindowDelay  = 1500;
    // Frame
    int frameThickness = 7;
    // Commands
    std::string newWindowCommand = "xterm";
    bool execUsingShell = false;

    // Load config: defaults -> system config -> user config -> CLI overrides
    static Config load(int argc, char** argv);

    // Apply a config file (only sets keys present in the file)
    void applyFile(const std::string& path);

    // Apply a single key=value pair
    void applyKeyValue(const std::string& key, const std::string& value);

    // Apply CLI arguments (getopt_long)
    void applyCliArgs(int argc, char** argv);
};
```

Create src/Config.cpp with implementation:

1. #define _GNU_SOURCE at top (per Pitfall 5 in RESEARCH.md)
2. #include "Config.h", <getopt.h>, <cstdlib>, <cstdio>, <fstream>, <sstream>, <algorithm>
3. xdgConfigHome() static function:
   - Read $XDG_CONFIG_HOME via std::getenv
   - If set and starts with '/', return it
   - Else read $HOME, fallback to "/tmp"
   - Return "$HOME/.config"
4. xdgConfigDirs() static function:
   - Read $XDG_CONFIG_DIRS via std::getenv
   - Parse colon-separated with std::getline(ss, token, ':')
   - Skip relative paths (must start with '/')
   - Default to {"/etc/xdg"} if empty
5. Config::applyFile(path):
   - Open std::ifstream(path), return silently if not openable (per Pitfall 7)
   - Loop over lines, skip blank lines and lines starting with '#'
   - Split on first '=' to get key/value, trim whitespace
   - Skip lines > 4096 chars with warning
   - Reject values > 256 chars with warning
   - Call applyKeyValue(key, value) for each valid line
   - Warn on lines without '=' but continue parsing
6. Config::applyKeyValue(key, value):
   - Match key string to member variable
   - For bool settings: accept "true"/"false", "1"/"0"
   - For int settings: parse with std::stoi, clamp to ranges:
     - autoRaiseDelay: 1-60000
     - pointerStoppedDelay: 1-60000
     - destroyWindowDelay: 1-60000
     - frameThickness: 1-50
   - For string settings: assign directly (color validation happens at X11 allocation time per RESEARCH.md Pitfall 2)
   - Warn on unknown keys but do not abort
7. Config::load(argc, argv):
   - Create Config with defaults
   - For each dir in xdgConfigDirs(): applyFile(dir + "/wm2-born-again/config")
   - applyFile(xdgConfigHome() + "/wm2-born-again/config")
   - applyCliArgs(argc, argv) -- stub that does nothing for now (Plan 02 implements this)
   - Return config

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| config file on disk -> Config struct | Untrusted input: user-edited text file. Malformed lines, invalid values, path traversal attempts. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | applyFile() path construction | mitigate | Fixed subdirectory "wm2-born-again/config" appended to XDG base; no user-supplied path components in file path construction |
| T-05-02 | Denial of Service | applyFile() line parsing | mitigate | Lines > 4096 chars skipped with warning; values > 256 chars rejected with warning. Prevents memory exhaustion from malicious config files |
| T-05-03 | Denial of Service | applyKeyValue() int parsing | mitigate | Integer values clamped to valid ranges: delays 1-60000ms, thickness 1-50px. Prevents extreme values causing WM hangs |
| T-05-04 | Denial of Service | applyKeyValue() std::stoi | mitigate | Wrap std::stoi in try/catch for std::invalid_argument and std::out_of_range. Warn and skip on parse failure |
| T-05-05 | Elevation of Privilege | new-window-command config key | accept | Config runs as the WM user; command injection is not an escalation since the user already has shell access. Per XDG model, user config is trusted |
</threat_model>
