**All three headers follow these modernization rules (per D-08, BLD-02):**
- `Boolean` -> `bool`, `True`/`False` -> `true`/`false`
- `ClientList` (macro) -> `std::vector<Client*>` (Phase 3 switches to `std::unique_ptr<Client>`)
- `char*` / `const char*` for strings -> `std::string`
- Raw `Display*` -> `x11::DisplayPtr` in Manager
- Raw `GC` -> `x11::GCPtr` where Manager/Border owns the GC
- Raw `XFontStruct*` -> `x11::FontStructPtr`
- Raw `Cursor` -> `x11::UniqueCursor` where Manager owns cursors
- `Window` stays raw per D-06 (it is an XID integer)
- `m_` prefix retained for member variables
- Use `#pragma once` for header guards
- Forward declarations for circular deps: `class Client;` in Manager.h, `class Client; class WindowManager;` in Border.h

**2. src/Manager.cpp** -- The core. Rewrite upstream Manager.C in C++17. Key functions:

Constructor follows upstream pattern (Manager.C:27-117):
- Print banner to stderr
- Open display via `x11::DisplayPtr(XOpenDisplay(nullptr))`, fatal if null
- Set error handler via `XSetErrorHandler(errorHandler)`
- Set up signal handlers via `sigaction()` (NOT the upstream signal() macro -- call sigaction directly)
- Intern all atoms (wm_state, wm_changeState, wm_protocols, wm_delete, wm_takeFocus, wm_colormaps, wm2_running)
- Check Shape extension via `XShapeQueryExtension` -- print warning if missing but DO NOT exit (graceful fallback per CONCERNS.md and Pitfall 5)
- Call `initialiseScreen()` -- get screen number, root window, default colormap, allocate colors (hard-coded from Config.h defaults), load fonts, create cursors, create GC
- Claim WM selection via `_WM2_RUNNING` atom on root window (sets _WM2_RUNNING property, checks for existing WM)
- Set SubstructureRedirectMask | SubstructureNotifyMask on root window
- Call `scanInitialWindows()` -- iterate existing top-level windows, create Client for each
