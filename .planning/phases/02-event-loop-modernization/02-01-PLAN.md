The new `nextEvent()`:
```cpp
void WindowManager::nextEvent(XEvent *e)
{
    struct pollfd fds[2];
    fds[0].fd = ConnectionNumber(display());
    fds[0].events = POLLIN;
    fds[1].fd = m_pipeRead.get();
    fds[1].events = POLLIN;

    while (m_looping) {

        // Check Xlib's internal queue first (Xlib may have buffered events)
        if (QLength(display()) > 0) {
            XNextEvent(display(), e);
            return;
        }

        // Flush pending X output before blocking
        XFlush(display());

        int timeout = -1;  // block indefinitely by default
        if (m_focusChanging) {
            timeout = 20;   // check focus delays every 20ms (matches upstream select granularity)
        }

        int r = poll(fds, 2, timeout);

        if (r < 0) {
            if (errno == EINTR) continue;  // signal interrupted, re-check m_looping
            std::perror("wm2: poll failed");
            m_looping = false;
            m_returnCode = 1;
            return;
        }

        // Signal pipe readable? (signal handler wrote a byte)
        if (fds[1].revents & POLLIN) {
            // Drain pipe (handler may have written multiple bytes)
            char buf[32];
            while (read(m_pipeRead.get(), buf, sizeof(buf)) > 0) { /* drain */ }
            std::fprintf(stderr, "wm2: signal caught, exiting\n");
            m_looping = false;
            m_returnCode = 0;
            return;
        }

        // Timer expired (r == 0 means timeout, no fd ready)
        if (r == 0) {
            if (m_focusChanging) {
                checkDelaysForFocus();
            }
            continue;  // re-check X11 queue, then poll again
        }

        // X11 fd readable
        if (fds[0].revents & POLLIN) {
            XNextEvent(display(), e);
            return;
        }
    }
}
```

Key behavioral properties of this implementation:
- **No select()**: Uses poll() with exactly 2 fds (X11 connection + self-pipe read end)
- **No goto**: Clean while loop with continue/return control flow
- **Xlib queue check first**: Prevents missing buffered events (Pitfall 2 from research)
- **Self-pipe drain**: Reads all available bytes, not just one (standard self-pipe pattern)
- **EINTR handling**: Re-enters loop to re-check m_looping (signal may have set it)
- **Focus timeout**: 20ms granularity matching upstream's select() timeout (checkDelaysForFocus stub is called but does nothing until Plan 02)
- **Loop unchanged**: The `loop()` dispatch switch (lines 9-116) is NOT modified

Note: The poll timeout uses a fixed 20ms when `m_focusChanging` is true, matching the upstream's select() granularity. Plan 02 will refine this to use chrono deadlines for the two-phase auto-raise timer, but for this plan the behavior is functionally identical to the upstream.

| Boundary | Description |
|----------|-------------|
| POSIX signal -> signal handler | Kernel delivers signal to process; handler must only call async-signal-safe functions |
| Signal handler -> self-pipe | Handler writes byte to pipe; pipe is the only safe IPC from signal context |
| Event loop -> X11 server | X11 protocol messages read from socket; poll() detects readable state |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-02-01 | Tampering | sigHandler() | mitigate | Handler only calls write() + volatile assignment (async-signal-safe per POSIX Table 2-4). No X11 calls, no heap access, no mutex. |
| T-02-02 | Denial of Service | self-pipe write end | mitigate | O_NONBLOCK on write end prevents handler deadlock if pipe buffer fills (Pitfall 1 from research) |

| T-02-04 | Denial of Service | poll() return values | mitigate | EINTR handled (re-enter loop), other errors set m_looping=false for clean exit |
